# 公众号每日运营

对已配置的微信公众号账号执行本地每日运营流程。

## 产品边界

- 优先做只读状态检查和推荐，再决定是否触发后台同步。
- 不要和既有定时同步冲突；完整同步前先检查今天是否已经跑过。
- 用户没有明确要求时，不上传草稿；上传交给 `woa-draft-upload`。
- 不发布、不群发。

## 工作流

1. 检查账号状态：

```bash
woa doctor
woa account list
woa status
```

2. 查看今天的素材和草稿：

```bash
woa material list --date today
woa draft list
```

3. 如果用户要判断今天发什么，先基于素材列表和已有草稿给出建议。

4. 如果用户明确要求同步，并且状态显示今天还没有成功同步：

```bash
woa sync
```

5. 如果用户要求生成候选草稿：

```bash
woa draft generate
```

## 推荐标准

- 优先推荐原创、实用、对中文读者有增量的信息。
- 同时考虑账号定位、近期话题重复度、阅读/涨粉/收益趋势和素材新鲜度。
- 高阅读但收益为 0 时，先按微信结算异常看待，不自动判定为差内容。
- 谣言或非一手来源要标记为需要核验。

## 完成汇报

汇报账号状态、今日素材、已有草稿、推荐内容，以及同步或登录问题。下一步动作要说清楚。
